Estilos de escrita CSS

BEM (Block, Element, Modifier) é um estilo de escrita de CSS que se baseia em nomes de classes para identificar diferentes tipos de elementos em um site. As classes são compostas por três partes: um bloco, um elemento e um modificador. Os blocos são elementos independentes e reutilizáveis, como por exemplo, um botão, uma barra de navegação ou um card. Eles são identificados por um nome único, precedido de um traço duplo (--). Por exemplo, "header--navbar" é um bloco que representa uma barra de navegação no cabeçalho. Além disso, os nomes das classes são escritos em letras minúsculas e separadas por traços. Dessa forma, é possível identificar facilmente quais são os elementos e modificadores de um bloco. Isso facilita a manutenção do código e a reutilização de estilos.

OOCSS (Object-Oriented CSS) é um estilo de escrita que se baseia na criação de objetos reutilizáveis. Isso é conseguido através da separação dos estilos em classes independentes e a utilização de herança. A ideia principal do OOCSS é criar objetos estilísticos independentes, chamados de "objetos" que podem ser aplicados a diferentes elementos da página. Esses objetos são criados através de classes CSS que podem ser reutilizadas em outros elementos. Por exemplo, um objeto "botao" pode ser criado com as classes "botao" e "botao-principal" e pode ser aplicado a vários botões diferentes na página. Isso permite que o estilo seja aplicado de forma consistente em todos os botões, sem precisar escrever o código CSS para cada botão individualmente.

Dry CSS é um estilo de escrita que busca evitar a duplicação de código e aumentar a legibilidade do código. Isso é conseguido através da utilização de herança, mixins e variáveis.

O estilo mais utilizado é o BEM, pois ele é simples de entender e facilita a manutenção do código. Além disso, ele é amplamente aceito na comunidade de desenvolvimento web e é usado por muitas empresas de desenvolvimento de software.
